# OpenCode Feature Matrix

This document maps the current low-level OpenCode implementation across the CLI, daemon, UI, and test surfaces. It is intended as an implementation inventory for the future unified architecture, not as end-user documentation.

## Status legend

- `supported`: primary current path
- `partial`: works, but only on one runtime path or with notable gaps
- `legacy`: still wired, but no longer the preferred path
- `unsupported`: intentionally absent today

## Cross-cutting migration themes

1. OpenCode still has two materially different runtime paths: server-native and ACP. The unified architecture should make transport an implementation detail, not a user-visible metadata split.
2. Several metadata shapes are still transport-branded even when the capability is not (`acpSessionModesV1`, `acpSessionModelsV1`, `directSessionV1`, `opencodeResumeHistoryImportV1`), although server mode now also publishes transport-neutral `sessionModesV1`.
3. Resume, direct sessions, takeover, and handoff already preserve much more of the same affinity tuple via the canonical runtime descriptor (`backendMode`, explicit server URL, remote session id, transcript source), but compatibility fallback fields still remain.
4. Execution runs are no longer ACP-only; they now default to the server-native adapter. The remaining mismatch is cleanup and parity hardening rather than missing source support.

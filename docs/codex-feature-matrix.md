# Codex Feature Matrix

This document is an implementation inventory for the current low-level Codex integration across the CLI, daemon, UI, protocol metadata, and test surfaces. It is intentionally architecture-facing and focuses on the code paths that exist today, the transport splits that still leak through, and the migration seams that should collapse into a unified provider session model.

## Status legend

- `supported`: primary or fully wired path
- `partial`: works, but only on some transports or with notable gaps
- `legacy`: still wired, but no longer the preferred path
- `unsupported`: intentionally absent today

## Cross-cutting migration themes

1. Codex still has three live runtime paths with materially different capability envelopes: app-server (richest), ACP (compatibility + installable), and MCP (fallback). The architecture should preserve the transports but stop making the rest of the product reason about them directly.
2. Resume, wake, direct sessions, and handoff all reconstruct the same identity tuple in different ways: `codexSessionId`, `codexBackendMode`, `CODEX_HOME`, transcript storage, working directory, and runtime descriptor/provider-extra payloads. That tuple is now substantially serialized, but some compatibility fields still remain and should continue collapsing into the canonical provider runtime descriptor.
3. App-server already publishes transport-neutral session-control metadata (`sessionModesV1`, `sessionModelsV1`, `sessionConfigOptionsV1`), and handoff import now preserves canonical Codex runtime affinity without `experimentalCodexAcp`; some spawn/resume paths still carry ACP-era compatibility fields.
4. Execution runs are no longer ACP/MCP-only; they now have an app-server path and prefer it by default when available. The remaining work is parity polish and cleanup of the fallback seams.
5. Direct-session takeover and handoff imports are now much closer: both preserve backend/runtime affinity. The remaining stability question is healthy-stack proof rather than missing source wiring.

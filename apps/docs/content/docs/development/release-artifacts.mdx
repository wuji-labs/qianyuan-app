## Channels

- `stable`
- `preview`
- `dev`

## Artifact naming

- CLI binary archives: `happier-v{version}-{os}-{arch}.tar.gz`
- hstack binary archives: `hstack-v{version}-{os}-{arch}.tar.gz`
- Self-host server runtime archives: `happier-server-v{version}-{os}-{arch}.tar.gz`
- Checksums: `checksums-{product}-v{version}.txt`
- Checksums signature: `checksums-{product}-v{version}.txt.minisig`

## Supported targets

CLI + hstack:

- `linux-x64`
- `linux-arm64`
- `darwin-x64`
- `darwin-arm64`
- `windows-x64`

Self-host server runtime:

- `linux-x64`
- `linux-arm64`
- `darwin-x64`
- `darwin-arm64`
- `windows-x64`

## Manifest schema (`/v1`)

Each platform manifest includes:

- `product`
- `channel`
- `version`
- `os`
- `arch`
- `url`
- `sha256`
- `signature`
- `publishedAt`
- `minSupportedVersion`
- `rolloutPercent`
- `critical`
- `notesUrl`
- `build.commitSha`
- `build.workflowRunId`

## Rollback policy

- CLI/hstack: rolling tags (`cli-stable` / `cli-preview` / `cli-dev`, `stack-stable` / `stack-preview` / `stack-dev`) + versioned tags.
- Self-host runtime: rolling server release tags + versioned artifacts.
- `happier self update --to <version>` is a version-targeted rollback entry point.
